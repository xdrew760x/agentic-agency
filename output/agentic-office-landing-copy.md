## 5. Features

### Real-Time 3D Visualization
Watch agents move through a fully rendered Babylon.js office. They walk to workstations, glow when active, gather for briefings, and return with results. This is not a diagram. It is a live system.

### Dynamic Speech and Thought Bubbles
Agents surface their reasoning as they work. You see the Researcher weighing data sources. The Copywriter testing headline options. The Debugger tracing a hook conflict. Visible thinking, not hidden inference.

### Task-Aware Behavior
Agents do not follow scripts. Their movement, dialogue, and status respond to real Claude Code sessions happening underneath. When a researcher is spawned, the Researcher walks to the desk and starts working. Real events. Real motion.

### Parallel Workflows
Multiple agents work simultaneously. The Copywriter drafts while the SEO Analyst maps keywords. The Debugger patches a conflict while the Code Reviewer waits for the diff. No bottleneck. No queue.

### Structured Delegation
Every task flows through the lead developer. Anders breaks work into pieces, assigns the right agent, and reviews the output. The hierarchy is clear. The chain of responsibility is visible.

### Persistent Memory
Agents share context across sessions. Client preferences, past decisions, research findings -- all stored in git-tracked memory files. No re-explaining. No lost context between conversations.

## 6. Technical Details

**Section Header:**
Under the hood

**Babylon.js 3D Engine**
Real-time WebGL rendering. Agent meshes with positional audio zones, glow layers, dynamic pathfinding, and camera systems that follow the action. Not a recording -- a live scene.

**Claude Code Orchestration**
The lead developer delegates via structured subagent prompts. Each agent runs as a scoped Claude Code instance with its own system prompt, tools, and output targets. Session hooks fire SSE events that drive the 3D layer.

**Server-Sent Events (SSE)**
A lightweight Node.js event server bridges Claude Code sessions to the browser. Agent activity -- task start, speech, movement, completion -- streams to the frontend in real time. No polling. No WebSocket overhead.

**Vite + Vanilla JS**
Zero-framework frontend. Vite for dev and build. Vanilla JavaScript driving Babylon.js. Fast cold starts. Small bundle. No React. No abstraction tax.

**MCP Server Integration**
Agents connect to WordPress, GitHub, and design tools through Model Context Protocol servers. The Scaffolder pushes pages. The Reviewer pulls diffs. Tools are used, not simulated.

## 7. FAQ

**Q: Is this a real working system or a concept demo?**
A: It is real. The 3D office visualizes actual Claude Code subagent sessions. When a researcher agent is spawned in the terminal, the Researcher avatar walks to their desk, glows, and surfaces real dialogue. The visualization layer is driven by live events, not pre-recorded animations.

**Q: What is an AI agent team?**
A: A group of specialized AI instances -- each with a defined role, tools, and scope -- coordinated by a lead agent or human operator. Instead of one general-purpose chatbot, you get a researcher, a copywriter, a debugger, and eight other specialists, each doing what they are built for.

**Q: Does this replace developers?**
A: No. Anders, the lead developer, delegates every task and reviews every deliverable. Agents handle execution -- research, first drafts, audits, scaffolding. The human sets strategy, makes judgment calls, and decides what ships.

**Q: What tech stack powers this?**
A: Babylon.js for 3D rendering. Claude Code (Opus 4) for agent orchestration. A Node.js SSE server for real-time event streaming. Vite for the frontend build. MCP servers for WordPress and GitHub integration.

**Q: Can I use this for my own agency or workflow?**
A: The architecture is modular. The agent definitions, event server, and 3D visualization are separate layers. If you run Claude Code with subagents and session hooks, you can wire up the same event-driven visualization for your own team structure.

**Q: How does the 3D visualization stay in sync with agent activity?**
A: Claude Code session hooks emit events on task start, agent spawn, speech, and completion. These events hit a local SSE server, which pushes them to the browser. The Babylon.js scene listens and translates each event into agent movement, glow states, and dialogue bubbles. Latency is under 100ms.
