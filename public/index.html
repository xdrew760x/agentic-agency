How the AI Agent Team Works

One task in. Finished work out. A multi-agent system where everything is visible in between.

01

Anders, the lead developer, receives your task and calls the relevant agents to the center table. The team gathers. The scope is set.

02

Under the hood

Technical Architecture

This is the stack behind an AI automation agency that runs in your browser.

3D Engine

Babylon.js

Real-time WebGL rendering. Agent meshes with positional audio zones, glow layers, dynamic pathfinding, and camera systems that follow the action. Not a recording — a live scene. babylonjs.com

Orchestration

Claude Code

The lead developer delegates via structured subagent prompts. Each agent runs as a scoped Claude Code instance with its own system prompt, tools, and output targets. Session hooks fire SSE events that drive the 3D layer. Anthropic docs

Real-Time

Server-Sent Events

A lightweight Node.js event server bridges Claude Code sessions to the browser. Agent activity — task start, speech, movement, completion — streams to the frontend in real time. No polling. No WebSocket overhead.

Frontend

Vite + Vanilla JS

Zero-framework frontend. Vite for dev and build. Vanilla JavaScript driving Babylon.js. Fast cold starts. Small bundle. No React. No abstraction tax.

Integration

MCP Servers

Agents connect to WordPress, GitHub, and design tools through Model Context Protocol servers. The Scaffolder pushes pages. The Reviewer pulls diffs. Tools are used, not simulated.
